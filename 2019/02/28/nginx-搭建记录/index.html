    2.keeplived
    1.简介：可配置管理LVS，同时可以对LVS下面的节点进行健康检测，也可实现网络服务的高可用功能，用来防止单点故障。
2.工作原理：
    协议：VRRP 全称Virtual Router Redundancy Protocol 虚拟路由冗余协议，即将多台功能相同的路由器组成一个路由器组，组内有一个master和多个backup，master上有一个对外提供服务的vip（虚拟ip），master组播vrrp包，通知backup master状态，当backup接收不到vrrp包后，根据vrrp的优先级选举master。来保证路由的高可用，业务的连续。    
3.补充：优点也是缺点。和zookeeper的高可用相比，简单，不需要业务层做任何事情，就可实现主备容灾，灾宕恢复的时间也比较短，
4.安装
    .环境预配置 
        10.6.136.31
        10.6.136.32
    .下载解压:keepalived-2.0.19.tar.gz
    .编译安装到指定目录：./configure --prefix=/usr/local/keepalived && make && make install    
    .cp /usr/local/keepalived/etc/init.d/keepalived /etc/init.d/
    .mkdir /etc/keepalived
    .cp /usr/local/keepalived/etc/keepalived/keepalived.conf /etc/keepalived/
    .cp /usr/local/keepalived/keepalived/etc/sysconfig/keepalived /etc/sysconfig/
    .cp /usr/local/keepalived/sbin/keepalived /usr/sbin/
    .systemctl enable keepalived 开启自启动
    .systemctl start keepalived or 去sbin目录下启动
    .tail -f  /var/log/messages 查看日志
